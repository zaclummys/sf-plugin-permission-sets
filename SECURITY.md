# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Instead, report them privately through GitHub's [private vulnerability reporting](https://github.com/zaclummys/sf-plugin-permission-sets/security/advisories/new).
If you cannot use that, email zaclummys@gmail.com with the details.

Please include as much of the following as you can:

- The affected version and command
- A description of the issue and its impact
- Steps to reproduce, or a proof of concept
- Any suggested remediation

You can expect an initial response within a few business days. We will keep you
informed as the issue is triaged and fixed, and we will credit you in the release
notes unless you prefer to remain anonymous.

## Scope

This plugin keeps its runtime dependencies minimal to reduce the overall attack
surface. Consumers should still have their security stakeholders review all
third-party products and their dependencies before adopting them.
